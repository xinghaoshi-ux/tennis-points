## 2. 优先级定义

| 优先级 | 含义 | 开发要求 |
|---|---|---|
| P0 | 主链路必需页面，缺少则闭环不通 | 必须在 MVP 中完成 |
| P1 | 辅助页面，提升管理效率但不阻塞主链路 | MVP 中完成，可简化 |

---

## 3. 页面与主链路的对应关系

### 3.1 主链路定义

MVP 主链路为：

```text
管理员登录 → 配置赛季 → 配置积分规则 → 维护选手 → 创建赛事 → 上传 Excel → 确认导入 → 生成积分 → 刷新排行榜 → 用户查看排行榜
```

### 3.2 页面在主链路中的位置

```text
主链路步骤                    对应页面
──────────                   ──────────
管理员登录                    A-01 登录
配置赛季                      A-03 赛季管理
配置积分规则                  A-06 积分规则配置
维护选手                      A-04 选手管理
创建赛事                      A-05 赛事管理
上传 Excel                    A-07 Excel 导入
确认导入                      A-07 Excel 导入
生成积分                      A-07 Excel 导入 / A-08 排行榜管理
刷新排行榜                    A-08 排行榜管理
用户查看排行榜                U-01 年度积分排行榜
```

### 3.3 主链路页面依赖关系

```text
A-01 登录
  ↓ （前置：无）
A-03 赛季管理
  ↓ （前置：需要先有 active 赛季）
A-06 积分规则配置
  ↓ （前置：需要先有 active 赛季）
A-04 选手管理
  ↓ （前置：无强依赖，但导入时需要选手数据）
A-05 赛事管理
  ↓ （前置：需要先有 active 赛季）
A-07 Excel 导入
  ↓ （前置：需要先有赛事、选手、积分规则）
A-08 排行榜管理
  ↓ （前置：需要先有积分数据）
U-01 年度积分排行榜
    （前置：需要排行榜已刷新）
```

## 4. 页面开发顺序建议

基于主链路依赖关系，建议按以下顺序开发：

### 4.1 第一批（框架与入口）

| 顺序 | 页面 | 原因 |
|---|---|---|
| 1 | A-01 登录 | 管理端入口，所有操作的前提 |
| 2 | AdminLayout | 管理端骨架，所有页面的容器 |

### 4.2 第二批（基础数据）

| 顺序 | 页面 | 原因 |
|---|---|---|
| 3 | A-03 赛季管理 | 赛季是所有业务数据的前提 |
| 4 | A-04 选手管理 | 选手是赛事结果的基础 |
| 5 | A-05 赛事管理 | 赛事是 Excel 导入的前提 |
| 6 | A-06 积分规则配置 | 规则是积分生成的前提 |

### 4.3 第三批（核心流程）

| 顺序 | 页面 | 原因 |
|---|---|---|
| 7 | A-07 Excel 导入 | 主链路核心操作 |
| 8 | A-08 排行榜管理 | 主链路终点（管理端） |

### 4.4 第四批（用户端）

| 顺序 | 页面 | 原因 |
|---|---|---|
| 9 | U-01 年度积分排行榜 | 主链路终点（用户端） |

### 4.5 第五批（辅助）

| 顺序 | 页面 | 原因 |
|---|---|---|
| 10 | A-02 仪表盘 | 非主链路，提升管理体验 |

## 5. 页面复杂度评估

| 页面 | 复杂度 | 核心难点 |
|---|---|---|
| U-01 排行榜 | 中 | 响应式布局（表格/卡片）、弹窗交互、搜索筛选 |
| A-01 登录 | 低 | 表单提交、Token 存储 |
| A-02 仪表盘 | 低 | 统计卡片、数据展示 |
| A-03 赛季管理 | 低 | 标准 CRUD |
| A-04 选手管理 | 低 | 标准 CRUD + 搜索筛选 |
| A-05 赛事管理 | 中 | 表单字段较多、状态展示 |
| A-06 积分规则 | 中 | 多 Tab、规则类型多样 |
| A-07 Excel 导入 | 高 | 步骤条、文件上传、异步轮询、预览表格、行级操作 |
| A-08 排行榜管理 | 中 | 异步刷新、积分明细弹窗 |

## 8. 验收标准

### 8.1 用户端验收

| 条件 | 标准 |
|---|---|
| 页面可访问 | 无需登录即可打开排行榜 |
| 数据展示 | 排行榜展示真实后端数据 |
| 搜索可用 | 输入姓名可过滤结果 |
| 筛选可用 | 选择院系可过滤结果 |
| 弹窗可用 | 点击姓名弹出积分明细 |
| 响应式 | 桌面端表格、移动端卡片均正常 |

### 8.2 管理端验收

| 条件 | 标准 |
|---|---|
| 登录可用 | 正确凭证可登录，错误凭证有提示 |
| 赛季可管理 | 创建、激活、关闭赛季正常 |
| 选手可管理 | 新增、编辑、搜索选手正常 |
| 赛事可管理 | 创建、编辑赛事正常 |
| 规则可配置 | 各类型积分规则可新增和编辑 |
| Excel 可导入 | 上传 → 解析 → 预览 → 确认全流程可用 |
| 排行榜可刷新 | 刷新后数据更新 |
| 主链路闭环 | 从创建赛季到用户端看到排名全流程跑通 |
