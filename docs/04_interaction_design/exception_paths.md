# 异常路径设计

> 本文定义 THA 年度赛事积分系统 MVP 阶段的异常路径，包括失败、超时、解析失败、提交失败、接口错误、人工干预等场景。
>
> 异常路径设计是前端错误提示、后端错误处理和 API 错误码设计的共同依据。

---

## 1. 异常分类

| 类别 | 说明 | 典型场景 |
|---|---|---|
| 网络异常 | 前后端通信失败 | 接口超时、网络断开、服务不可用 |
| 业务校验失败 | 数据不满足业务规则 | 字段缺失、格式错误、规则冲突 |
| 系统处理失败 | 后台任务执行出错 | Excel 解析失败、积分生成失败 |
| 权限异常 | 用户无权执行操作 | Token 过期、权限不足 |
| 数据冲突 | 操作与现有数据矛盾 | 重复导入、并发修改 |

## 2. 用户端异常路径

### 2.1 排行榜加载失败

| 维度 | 说明 |
|---|---|
| 触发条件 | 请求排行榜接口超时或返回错误 |
| 用户看到什么 | 空状态页面，提示"数据加载失败" |
| 可执行操作 | 点击"重试"按钮重新请求 |
| 系统行为 | 重新发起接口请求 |
| 重试策略 | 用户手动重试，无自动重试 |

### 2.2 选手积分明细加载失败

| 维度 | 说明 |
|---|---|
| 触发条件 | 请求选手积分明细接口失败 |
| 用户看到什么 | 弹窗内显示"加载失败"提示 |
| 可执行操作 | 关闭弹窗后重新点击选手姓名 |
| 系统行为 | 重新发起接口请求 |

### 2.3 无数据状态

| 维度 | 说明 |
|---|---|
| 触发条件 | 当前赛季无积分数据，或筛选条件无匹配结果 |
| 用户看到什么 | 空状态提示"暂无排名数据"或"未找到匹配选手" |
| 可执行操作 | 清除筛选条件、等待管理员录入数据 |

## 3. 管理端登录异常

### 3.1 登录失败

| 维度 | 说明 |
|---|---|
| 触发条件 | 用户名或密码错误 |
| 用户看到什么 | 表单下方提示"用户名或密码错误" |
| 可执行操作 | 重新输入凭证 |
| 系统行为 | 不透露具体是用户名错误还是密码错误 |

### 3.2 Token 过期

| 维度 | 说明 |
|---|---|
| 触发条件 | 管理员操作时 Token 已过期 |
| 用户看到什么 | 全局提示"登录已过期，请重新登录"，自动跳转登录页 |
| 可执行操作 | 重新登录 |
| 系统行为 | 清除本地 Token，重定向到登录页 |
| 数据保护 | 未提交的表单数据不做自动保存（MVP 阶段） |

## 4. Excel 上传异常路径

### 4.1 文件格式错误

| 维度 | 说明 |
|---|---|
| 触发条件 | 上传的文件不是 .xlsx 或 .xls 格式 |
| 用户看到什么 | 上传区域提示"仅支持 .xlsx 和 .xls 格式" |
| 可执行操作 | 重新选择正确格式的文件 |
| 系统行为 | 前端拦截，不发送请求 |
| 状态变化 | 无状态变化 |

### 4.2 文件过大

| 维度 | 说明 |
|---|---|
| 触发条件 | 文件大小超过系统限制（建议 10MB） |
| 用户看到什么 | 提示"文件大小超过限制，请控制在 10MB 以内" |
| 可执行操作 | 精简文件内容后重新上传 |
| 系统行为 | 前端拦截或后端返回 413 |
| 状态变化 | 无状态变化 |

### 4.3 上传网络中断

| 维度 | 说明 |
|---|---|
| 触发条件 | 文件上传过程中网络断开 |
| 用户看到什么 | 上传进度条停止，提示"上传失败，请检查网络后重试" |
| 可执行操作 | 检查网络后重新上传 |
| 系统行为 | 清理未完成的上传记录（如有） |
| 状态变化 | 无 Upload 记录产生 |

## 5. Excel 解析异常路径

### 5.1 解析超时

| 维度 | 说明 |
|---|---|
| 触发条件 | 后台解析任务超过最大执行时间（建议 60 秒） |
| 用户看到什么 | 步骤条"解析"步骤标红，提示"解析超时" |
| 可执行操作 | 查看超时原因、重新上传简化后的文件 |
| 系统行为 | 任务标记为 parse_failed，记录超时错误日志 |
| 状态变化 | Upload: parsing → parse_failed |

### 5.2 Excel 内容为空

| 维度 | 说明 |
|---|---|
| 触发条件 | Excel 文件无有效数据行 |
| 用户看到什么 | 提示"文件中未找到有效数据" |
| 可执行操作 | 检查文件内容后重新上传 |
| 系统行为 | 标记为 parse_failed |
| 状态变化 | Upload: parsing → parse_failed |

### 5.3 表头无法识别

| 维度 | 说明 |
|---|---|
| 触发条件 | Excel 表头与标准模板不匹配，系统无法确定字段含义 |
| 用户看到什么 | 提示"表头格式不符合标准模板，请使用标准模板重新上传" |
| 可执行操作 | 下载标准模板、按模板整理数据后重新上传 |
| 系统行为 | 标记为 parse_failed，记录具体不匹配的列 |
| 状态变化 | Upload: parsing → parse_failed |

### 5.4 部分行校验失败

| 维度 | 说明 |
|---|---|
| 触发条件 | Excel 整体可解析，但部分行存在数据问题 |
| 用户看到什么 | 导入预览中异常行标红，显示具体错误原因 |
| 可执行操作 | 修改选手匹配、忽略异常行、下载错误报告、取消导入 |
| 系统行为 | 解析成功，进入 parsed 状态，异常行标记在预览中 |
| 状态变化 | Upload: parsing → parsed（部分行有警告） |

常见行级错误：

| 错误类型 | 说明 | 处理方式 |
|---|---|---|
| 选手未匹配 | 姓名在系统中不存在 | 管理员手动匹配或新建选手 |
| 成绩不合法 | 成绩值不在允许范围内 | 管理员修正或忽略该行 |
| 重复数据 | 同一赛事同一选手同一成绩已存在 | 提示重复，管理员决定是否忽略 |
| 组别不合法 | 组别名称不在系统配置中 | 管理员修正或先配置组别 |
| 双打组合不完整 | 双打赛事只有一名选手 | 管理员补充搭档信息 |
| 团体队员为空 | 团体赛未提供有效队员名单 | 管理员补充队员信息 |

## 6. 导入确认异常路径

### 6.1 确认导入时接口失败

| 维度 | 说明 |
|---|---|
| 触发条件 | 管理员点击确认导入，后端写入失败 |
| 用户看到什么 | 提示"导入失败，请重试" |
| 可执行操作 | 重新点击确认导入 |
| 系统行为 | 事务回滚，Upload 状态保持 parsed |
| 状态变化 | 无变化，保持 parsed |

### 6.2 积分规则缺失

| 维度 | 说明 |
|---|---|
| 触发条件 | 导入的赛事结果对应的积分规则未配置 |
| 用户看到什么 | 导入预览中相关行提示"缺少对应积分规则" |
| 可执行操作 | 先去配置积分规则，再回来确认导入 |
| 系统行为 | 在预览阶段检测规则完整性，缺失时阻止确认 |
| 状态变化 | 保持 parsed，不允许进入 imported |

### 6.3 并发导入冲突

| 维度 | 说明 |
|---|---|
| 触发条件 | 同一赛事有多个管理员同时尝试导入 |
| 用户看到什么 | 提示"该赛事已有导入操作进行中，请稍后重试" |
| 可执行操作 | 等待后重试 |
| 系统行为 | 通过数据库锁或状态检查防止重复写入 |

## 7. 积分生成异常路径

### 7.1 积分生成失败

| 维度 | 说明 |
|---|---|
| 触发条件 | 积分计算过程中出现异常（规则匹配失败、数据不一致等） |
| 用户看到什么 | 提示"积分生成失败"，显示失败原因 |
| 可执行操作 | 查看错误详情、修正数据后重试 |
| 系统行为 | 事务回滚，不写入部分积分记录 |
| 状态变化 | 积分生成状态: processing → failed；Tournament 保持 completed |

### 7.2 团体赛分摊人数为零

| 维度 | 说明 |
|---|---|
| 触发条件 | 团体赛有效队员人数为 0，导致除零错误 |
| 用户看到什么 | 积分生成失败，提示"团体赛有效队员人数不能为零" |
| 可执行操作 | 修正团体队员信息后重试 |
| 系统行为 | 在计算前校验分摊人数，为零时拒绝生成 |

### 7.3 积分生成超时

| 维度 | 说明 |
|---|---|
| 触发条件 | 积分生成任务超过最大执行时间 |
| 用户看到什么 | 提示"积分生成超时，请重试" |
| 可执行操作 | 重试生成 |
| 系统行为 | 任务标记为 failed，事务回滚 |

## 9. 数据修正与撤回路径

### 9.1 撤回已发布赛事

| 维度 | 说明 |
|---|---|
| 触发条件 | 管理员发现已发布赛事的结果有误 |
| 操作流程 | 管理员在赛事详情页点击"撤回发布" → 确认撤回 |
| 系统行为 | 删除该赛事关联的 entries_points 记录，Tournament 回退为 completed |
| 后续操作 | 修正赛事结果 → 重新生成积分 → 重新发布 → 刷新排行榜 |
| 状态变化 | Tournament: published → completed；Ranking: up_to_date → stale |

### 9.2 修正已导入的赛事结果

| 维度 | 说明 |
|---|---|
| 触发条件 | 管理员发现已导入的赛事结果数据有误 |
| 前置条件 | 赛事必须先撤回到 completed 状态 |
| 操作流程 | 撤回发布 → 修改 event_results 记录 → 重新生成积分 → 重新发布 |
| 系统行为 | 提供赛事结果编辑界面，修改后重新走积分生成流程 |

## 10. 人工介入节点汇总

| 节点 | 触发场景 | 管理员可执行操作 | 操作后状态变化 |
|---|---|---|---|
| 导入预览确认 | Excel 解析完成 | 确认导入 / 修改匹配 / 忽略行 / 取消 | parsed → imported 或 cancelled |
| 选手匹配 | 解析结果中选手未匹配 | 手动选择已有选手 / 新建选手 | 预览行状态从"待匹配"变为"正常" |
| 积分生成确认 | 赛事结果导入后 | 触发积分生成 | Tournament: completed → published |
| 排行榜刷新 | 新积分数据产生后 | 手动刷新排行榜 | Ranking: stale → refreshing → up_to_date |
| 撤回发布 | 发现数据错误 | 确认撤回 | Tournament: published → completed |
| 关闭赛季 | 赛季结束 | 确认关闭 | Season: active → closed |

## 11. 错误提示规范

### 11.1 前端错误提示层级

| 层级 | 展示方式 | 适用场景 |
|---|---|---|
| 全局提示 | 顶部 Toast / Notification | 网络错误、Token 过期、操作成功 |
| 页面级提示 | 页面内 Alert 区域 | 数据加载失败、无数据状态 |
| 表单级提示 | 字段下方红色文字 | 输入校验失败 |
| 行级提示 | 表格行标红 + 错误说明列 | 导入预览中的异常行 |

### 11.2 错误信息原则

- 告诉用户发生了什么，而不是技术细节。
- 告诉用户可以做什么来解决问题。
- 不暴露系统内部错误堆栈。
- 可操作的错误提供操作按钮（重试、返回、联系管理员）。

## 12. 超时与重试策略

| 操作 | 超时时间 | 重试方式 | 最大重试次数 |
|---|---|---|---|
| 排行榜接口请求 | 10 秒 | 用户手动重试 | 无限制 |
| 选手积分明细请求 | 10 秒 | 用户手动重试 | 无限制 |
| Excel 上传 | 60 秒 | 用户手动重新上传 | 无限制 |
| Excel 解析任务 | 60 秒 | 系统标记失败，用户重新上传 | 不重试原任务 |
| 积分生成任务 | 30 秒 | 管理员手动重试 | 3 次 |
| 排行榜刷新 | 30 秒 | 管理员手动重试 | 无限制 |
| 管理端接口请求 | 15 秒 | 用户手动重试 | 无限制 |

## 13. 异常路径与状态迁移的关系

异常路径不会产生"悬空状态"。每个异常都有明确的状态归宿：

| 异常场景 | 最终状态 | 是否可恢复 |
|---|---|---|
| Excel 解析超时 | Upload: parse_failed | 不可恢复，需重新上传 |
| Excel 部分行错误 | Upload: parsed（带警告） | 可恢复，管理员修正后确认 |
| 确认导入失败 | Upload: parsed | 可恢复，重试确认 |
| 积分生成失败 | Tournament: completed | 可恢复，修正后重试 |
| 排行榜刷新失败 | Ranking: stale | 可恢复，重试刷新 |
| Token 过期 | Auth: expired | 可恢复，重新登录 |
