# 用户主流程设计

> 本文定义 THA 年度赛事积分系统 MVP 阶段的用户主流程。
>
> 系统有两类用户角色：普通用户（公共访客）和管理员。两类角色的主流程独立但数据相通。

---

## 1. 普通用户主流程

普通用户无需登录，进入系统即为公共积分榜查看者。

### 1.1 流程总览

```text
进入系统
  ↓
看到年度积分排行榜
  ↓
浏览排行榜列表
  ↓
（可选）搜索选手姓名
  ↓
（可选）按院系筛选
  ↓
点击选手姓名
  ↓
弹出选手积分明细
  ↓
查看积分构成与赛事结果
  ↓
关闭弹窗，继续浏览
```

### 1.2 逐步拆解

#### 步骤 1：进入系统

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 加载中状态，随后展示排行榜页面 |
| 用户可以做什么 | 等待页面加载完成 |
| 系统会做什么 | 请求当前赛季信息，请求排行榜数据 |

#### 步骤 2：查看排行榜

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 赛季名称、排行榜列表（排名、积分、姓名、年龄、参赛数量、院系） |
| 用户可以做什么 | 上下滚动浏览、搜索、筛选、点击选手姓名 |
| 系统会做什么 | 展示排行榜数据，同积分并列排名，按姓名首字母排序同分选手 |

#### 步骤 3：搜索与筛选

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 搜索框、院系筛选器、筛选后的排行榜结果 |
| 用户可以做什么 | 输入姓名关键字、选择院系、清除筛选条件 |
| 系统会做什么 | 根据条件过滤排行榜数据并重新展示 |

#### 步骤 4：查看选手积分明细

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 弹窗展示选手基础信息、积分统计、积分明细列表 |
| 用户可以做什么 | 浏览积分来源、查看各赛事获得积分、关闭弹窗 |
| 系统会做什么 | 请求该选手积分明细接口，按积分来源分类展示 |

## 2. 管理员主流程

管理员需要登录，负责赛事数据录入和积分管理。

### 2.1 流程总览

```text
登录管理后台
  ↓
确认当前赛季
  ↓
配置积分规则
  ↓
维护选手信息
  ↓
创建赛事
  ↓
上传赛事结果 Excel
  ↓
系统解析 Excel
  ↓
查看导入预览
  ↓
确认导入
  ↓
系统生成积分记录
  ↓
刷新排行榜
  ↓
用户端展示最新排名
```

### 2.2 逐步拆解

#### 步骤 1：登录管理后台

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 登录页面，包含用户名和密码输入框 |
| 用户可以做什么 | 输入凭证并提交登录 |
| 系统会做什么 | 验证凭证，成功后跳转仪表盘，失败则提示错误 |

#### 步骤 2：确认当前赛季

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 仪表盘顶部显示当前赛季名称和状态 |
| 用户可以做什么 | 查看赛季信息；如无当前赛季，进入赛季管理创建并激活 |
| 系统会做什么 | 加载当前激活赛季，若无激活赛季则提示管理员设置 |

#### 步骤 3：配置积分规则

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 积分规则配置页面，按赛事类型分 Tab 展示规则表 |
| 用户可以做什么 | 新增规则、编辑规则、启用或禁用规则 |
| 系统会做什么 | 保存规则配置，关联到当前赛季 |

#### 步骤 4：维护选手信息

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 选手列表页面，支持搜索和筛选 |
| 用户可以做什么 | 新增选手、编辑选手信息、合并重复选手 |
| 系统会做什么 | 保存选手基础信息到数据库 |

#### 步骤 5：创建赛事

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 赛事创建表单，包含名称、级别、组别、日期等字段 |
| 用户可以做什么 | 填写赛事信息并保存 |
| 系统会做什么 | 创建赛事记录，状态为 draft |

#### 步骤 6：上传赛事结果 Excel

| 维度 | 说明 |
|---|---|
| 用户看到什么 | Excel 导入页面，步骤条显示当前处于"上传"步骤 |
| 用户可以做什么 | 选择关联赛事、选择 Excel 文件、点击上传 |
| 系统会做什么 | 接收文件，创建上传记录，状态为 pending |

#### 步骤 7：系统解析 Excel

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 步骤条进入"解析"步骤，显示解析进度 |
| 用户可以做什么 | 等待解析完成；解析失败时可查看错误信息并重新上传 |
| 系统会做什么 | 后台任务读取 Excel，校验数据格式和内容，生成结构化结果 |

#### 步骤 8：查看导入预览

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 导入预览表格，每行显示赛事、成绩、选手、预计积分、状态 |
| 用户可以做什么 | 逐行检查、修改选手匹配、忽略异常行、下载错误报告 |
| 系统会做什么 | 展示解析结果，标记正常行和异常行 |

#### 步骤 9：确认导入

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 确认按钮和导入摘要（总行数、正常行数、忽略行数） |
| 用户可以做什么 | 点击确认导入，或返回修改 |
| 系统会做什么 | 将确认的数据写入 event_results 表 |

#### 步骤 10：系统生成积分记录

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 积分生成进度提示 |
| 用户可以做什么 | 等待生成完成 |
| 系统会做什么 | 根据积分规则为每位选手生成 entries_points 记录 |

#### 步骤 11：刷新排行榜

| 维度 | 说明 |
|---|---|
| 用户看到什么 | 排行榜管理页面，显示刷新按钮和最近刷新时间 |
| 用户可以做什么 | 点击刷新排行榜，查看最新排名 |
| 系统会做什么 | 重新计算年度排行榜，更新排名数据 |

## 3. 管理员核心操作闭环

以下是管理员最核心的操作闭环，即"从上传 Excel 到排行榜更新"：

```text
┌─────────────────────────────────────────────────────────────┐
│                    Excel 导入闭环                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  上传 Excel ──→ 解析 ──→ 预览 ──→ 确认 ──→ 生成积分 ──→ 刷新榜 │
│       │              │         │                              │
│       │              ↓         ↓                              │
│       │          解析失败    修改/忽略                          │
│       │              │         │                              │
│       ↓              ↓         ↓                              │
│    重新上传      查看错误    返回预览                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 4. 两类用户的数据关系

```text
管理员操作                              用户端展示
─────────────                          ─────────────
配置赛季          ──────────────→      赛季信息展示
配置积分规则      ──────────────→      积分规则说明
维护选手          ──────────────→      选手基础信息
创建赛事          ──────────────→      赛事列表（后续）
导入赛事结果      ──────────────→      积分明细数据
刷新排行榜        ──────────────→      排行榜展示
```

管理员的每一步操作最终都服务于用户端的数据展示。排行榜是两端的交汇点。

## 5. MVP 主流程边界

本文档仅覆盖 MVP 阶段的主流程，以下内容不在本文档范围内：

- AI 智能解析非标准 Excel
- Elo / Glicko 评分
- 逐场比分录入
- 社交功能
- 自动签表与赛程编排
- 多赛季对比
- 排名走势图
