# 前后端联调顺序

> 本文定义 THA 年度赛事积分系统前后端联调的推荐顺序、依赖关系和验收标准。
>
> 联调按"基础设施 → 数据准备 → 核心流程 → 展示层"的顺序推进。

---

## 1. 联调原则

| 原则 | 说明 |
|---|---|
| 先通后全 | 先跑通最小路径，再补全边界和异常 |
| 先后端后前端 | 后端接口就绪并通过 Postman/curl 验证后，前端再对接 |
| 先管理后公共 | 管理端产生数据，公共端消费数据 |
| 先同步后异步 | 同步接口先联调，异步流程（上传解析、积分生成）后联调 |

---

## 2. 联调阶段总览

```text
阶段 1：基础通路        → 健康检查 + 认证
阶段 2：数据管理        → 赛季 + 选手 + 赛事 + 积分规则
阶段 3：核心流程        → Excel 上传 → 解析 → 预览 → 确认 → 积分生成
阶段 4：数据展示        → 排行榜 + 积分明细
阶段 5：补全增强        → 仪表盘 + 撤回 + 取消 + 异常处理
```

---

## 3. 阶段 1：基础通路

目标：验证前后端可通信、认证机制正常。

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 1.1 | /api/v1/health | GET | 返回 200，status=ok |
| 1.2 | /api/v1/admin/auth/login | POST | 正确凭证返回 Token；错误凭证返回 401 |
| 1.3 | /api/v1/admin/auth/me | GET | 携带 Token 返回用户信息；无 Token 返回 401 |

前端验收：
- 管理端登录页可正常登录
- Token 存储到 localStorage
- 页面刷新后 Token 仍有效
- 无效 Token 自动跳转登录页

依赖：无

---

## 4. 阶段 2：数据管理

目标：管理端可完成基础数据的 CRUD。

### 4.1 赛季管理（优先）

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 2.1 | /api/v1/admin/seasons | POST | 创建赛季，status=draft |
| 2.2 | /api/v1/admin/seasons | GET | 列表展示所有赛季 |
| 2.3 | /api/v1/admin/seasons/{id} | PUT | 编辑赛季名称和日期 |
| 2.4 | /api/v1/admin/seasons/{id}/activate | POST | 激活赛季，原 active 变 closed |

前端验收：
- 赛季列表页正常展示
- 创建赛季表单提交成功
- 激活操作后状态正确变更

依赖：阶段 1 完成

### 4.2 选手管理

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 2.5 | /api/v1/admin/players | POST | 创建选手 |
| 2.6 | /api/v1/admin/players | GET | 列表展示，支持搜索和院系筛选 |
| 2.7 | /api/v1/admin/players/{id} | GET | 查看选手详情 |
| 2.8 | /api/v1/admin/players/{id} | PUT | 编辑选手信息 |

前端验收：
- 选手列表分页正常
- 搜索和筛选功能正常
- 创建/编辑表单校验正常

依赖：阶段 1 完成

### 4.3 赛事管理

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 2.9 | /api/v1/admin/tournaments | POST | 创建赛事，自动关联当前 active 赛季 |
| 2.10 | /api/v1/admin/tournaments | GET | 列表展示，支持状态和赛季筛选 |
| 2.11 | /api/v1/admin/tournaments/{id} | GET | 查看赛事详情 |
| 2.12 | /api/v1/admin/tournaments/{id} | PUT | 编辑赛事（仅 draft 状态） |

前端验收：
- 无 active 赛季时创建赛事返回明确错误
- 赛事列表筛选正常
- draft 状态可编辑，非 draft 状态编辑按钮禁用

依赖：赛季管理完成（需要 active 赛季）

### 4.4 积分规则管理

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 2.13 | /api/v1/admin/points-rules | POST | 创建规则 |
| 2.14 | /api/v1/admin/points-rules | GET | 列表展示，支持类型筛选 |
| 2.15 | /api/v1/admin/points-rules/{id} | PUT | 编辑规则 |
| 2.16 | /api/v1/admin/points-rules/{id} | DELETE | 删除规则（未被引用时） |

前端验收：
- 规则列表按类型分组展示
- 重复规则组合创建时返回冲突错误
- 已引用规则删除时提示不可删除

依赖：赛季管理完成（规则归属赛季）

---

## 5. 阶段 3：核心流程

目标：打通 Excel 导入 → 积分生成的完整链路。

### 5.1 Excel 上传与解析

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 3.1 | /api/v1/admin/uploads | POST | 上传 .xlsx 文件，返回 upload_id |
| 3.2 | /api/v1/admin/uploads/{id} | GET | 轮询状态变化：pending → parsing → parsed |
| 3.3 | /api/v1/admin/uploads/{id}/preview | GET | parsed 后返回逐行预览数据 |

前端验收：
- 文件选择后上传，显示进度
- 轮询期间显示"解析中"状态
- 解析完成后展示预览表格
- 异常行标红，显示错误原因
- 非 .xlsx 文件上传时前端拦截

依赖：赛事管理完成（上传关联赛事）、选手数据已录入（解析需匹配选手）

### 5.2 确认导入

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 3.4 | /api/v1/admin/uploads/{id}/confirm | POST | 提交确认行和忽略行，状态变为 imported |

前端验收：
- 管理员可勾选/取消行
- 确认后赛事状态变为 completed
- 确认后不可重复操作

依赖：5.1 完成

### 5.3 积分生成

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 3.5 | /api/v1/admin/tournaments/{id}/generate-points | POST | 返回 202，异步生成积分 |
| 3.6 | /api/v1/admin/tournaments/{id} | GET | 轮询状态：completed → published |

前端验收：
- 点击"生成积分"后显示处理中
- 生成完成后赛事状态变为 published
- 非 completed 状态时按钮禁用

依赖：5.2 完成、积分规则已配置

## 6. 阶段 4：数据展示

目标：公共端排行榜和积分明细正常展示。

### 6.1 公共接口

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 4.1 | /api/v1/public/seasons/current | GET | 返回当前 active 赛季 |
| 4.2 | /api/v1/public/departments | GET | 返回院系列表 |
| 4.3 | /api/v1/public/rankings | GET | 返回排行榜，支持分页、搜索、筛选 |
| 4.4 | /api/v1/public/players/{id}/points | GET | 返回选手积分明细和分类汇总 |

前端验收：
- 用户端排行榜页正常加载
- 搜索输入后实时筛选
- 院系下拉筛选正常
- 点击选手姓名弹出积分明细
- 积分分类汇总数据正确

依赖：阶段 3 完成（需要已生成的积分数据）

### 6.2 排行榜刷新

| 序号 | 接口 | 方法 | 验收标准 |
|---|---|---|---|
| 4.5 | /api/v1/admin/rankings/refresh | POST | 触发刷新，返回 202 |
| 4.6 | /api/v1/admin/rankings | GET | 管理端查看排行榜 |

前端验收：
- 管理端排行榜页正常展示
- 刷新按钮点击后数据更新

依赖：阶段 3 完成

## 7. 阶段 5：补全增强

目标：补全非核心功能和异常路径。

| 序号 | 接口 | 方法 | 优先级 | 验收标准 |
|---|---|---|---|---|
| 5.1 | /api/v1/admin/dashboard | GET | P1 | 返回统计概览数据 |
| 5.2 | /api/v1/admin/seasons/{id}/close | POST | P1 | 关闭赛季 |
| 5.3 | /api/v1/admin/uploads/{id}/cancel | POST | P1 | 取消上传 |
| 5.4 | /api/v1/admin/tournaments/{id}/revoke-publish | POST | P1 | 撤回发布，积分回退 |

前端验收：
- 仪表盘页展示统计数据
- 撤回发布后赛事回退为 completed，排行榜数据更新

依赖：阶段 4 完成

## 10. 联调检查清单

每个阶段完成后，逐项确认：

### 阶段 1 检查项
- [ ] curl /api/v1/health 返回 200
- [ ] 登录接口返回有效 Token
- [ ] Token 过期后返回 401
- [ ] 前端登录流程完整

### 阶段 2 检查项
- [ ] 赛季 CRUD 正常
- [ ] 赛季激活/关闭状态流转正确
- [ ] 选手 CRUD + 搜索筛选正常
- [ ] 赛事 CRUD + 状态约束正常
- [ ] 积分规则 CRUD + 唯一性约束正常

### 阶段 3 检查项
- [ ] Excel 上传成功，返回 upload_id
- [ ] 轮询状态变化正常（3 秒间隔）
- [ ] 预览数据展示正确，异常行标识清晰
- [ ] 确认导入后赛事状态变为 completed
- [ ] 积分生成后赛事状态变为 published
- [ ] 生成的积分记录数据正确

### 阶段 4 检查项
- [ ] 公共排行榜数据正确
- [ ] 排名并列逻辑正确（同积分同排名）
- [ ] 搜索和筛选响应正常
- [ ] 选手积分明细分类汇总正确
- [ ] 管理端排行榜与公共端一致

### 阶段 5 检查项
- [ ] 仪表盘统计数据正确
- [ ] 撤回发布后积分记录被删除
- [ ] 撤回后排行榜数据更新
- [ ] 取消上传状态正确变更
