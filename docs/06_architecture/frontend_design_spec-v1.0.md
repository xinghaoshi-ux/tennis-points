# 前端设计规范 v1.0

> 本文定义 THA 年度赛事积分系统 MVP 阶段的页面清单、页面结构、页面跳转关系和核心交互设计。
>
> 本文档是前端实施计划和 UI 设计的上游依据，不包含代码实现。

---

## 1. 页面清单总览

系统分为两个独立前端应用：用户端（公共积分榜）和管理端（Web 后台）。

### 1.1 用户端页面

| 编号 | 页面 | 路由 | 说明 |
|---|---|---|---|
| U-01 | 年度积分排行榜 | / | 用户端唯一页面，首页即排行榜 |

用户端不设独立的选手详情页，选手积分明细通过排行榜页面内的弹窗展示。

### 1.2 管理端页面

| 编号 | 页面 | 路由 | 说明 |
|---|---|---|---|
| A-01 | 登录 | /login | 管理员登录入口 |
| A-02 | 仪表盘 | /dashboard | 系统概览 |
| A-03 | 赛季管理 | /seasons | 赛季 CRUD |
| A-04 | 选手管理 | /players | 选手 CRUD |
| A-05 | 赛事管理 | /tournaments | 赛事 CRUD |
| A-06 | 积分规则配置 | /points-rules | 积分规则维护 |
| A-07 | Excel 导入 | /uploads | 赛事结果导入全流程 |
| A-08 | 排行榜管理 | /rankings | 查看和刷新排行榜 |

## 3. 页面结构层级

### 3.1 U-01 年度积分排行榜

```text
页面容器
├─ 顶部标题区
│  └─ 系统名称 + 赛季名称
├─ 筛选搜索区
│  ├─ 姓名搜索框
│  └─ 院系筛选器
├─ 排行榜列表区
│  ├─ 桌面端：表格布局（排名、积分、姓名、年龄、参赛数量、院系）
```

```text

│  ├─ 导入摘要
│  ├─ 确认按钮
│  └─ 返回修改按钮
└─ 步骤 5：导入完成
   ├─ 成功提示
```

## 4. 页面跳转关系

### 4.1 用户端

用户端为单页面应用，无页面间跳转。所有交互在排行榜页面内完成（弹窗展开/收起）。

### 4.2 管理端

```text
登录 ──[登录成功]──→ 仪表盘
                        │
         ┌──────────────┼──────────────────────────────┐
         ↓              ↓              ↓               ↓
      赛季管理       选手管理       赛事管理        积分规则
                                      │
                                      ↓
                                 Excel 导入
                                      │
                                      ↓
                                 排行榜管理
```

关键跳转路径：

| 起点 | 动作 | 终点 |
|---|---|---|
| 登录 | 登录成功 | 仪表盘 |
| 左侧菜单 | 点击菜单项 | 对应页面 |
| 赛事管理 | 点击"上传结果" | Excel 导入（预选赛事） |
| Excel 导入 | 导入完成后点击"生成积分" | 排行榜管理 |
| 排行榜管理 | 点击选手姓名 | 弹出积分明细弹窗 |
| 任意页面 | Token 过期 | 登录页 |

### 4.3 管理端 Layout 结构

```text
AdminLayout
├─ 顶部栏
│  ├─ 系统名称
│  ├─ 当前赛季
│  ├─ 管理员名称
│  └─ 退出登录
├─ 左侧菜单
│  ├─ 仪表盘
│  ├─ 赛季管理
│  ├─ 选手管理
│  ├─ 赛事管理
│  ├─ 积分规则
│  ├─ Excel 导入
│  └─ 排行榜管理
└─ 右侧内容区
   └─ 当前路由对应的页面
```

## 5. 页面与交互状态的对应关系

### 5.1 用户端

| 页面 | 对应状态 | 状态影响 |
|---|---|---|
| U-01 排行榜 | Ranking: up_to_date | 正常展示排行榜数据 |
| U-01 排行榜 | 接口请求失败 | 展示空状态 + 重试按钮 |
| U-01 积分明细弹窗 | 接口请求中 | 弹窗内显示加载动画 |
| U-01 积分明细弹窗 | 接口请求成功 | 展示积分明细数据 |

### 5.2 管理端

| 页面 | 对应状态 | 状态影响 |
|---|---|---|
| A-01 登录 | Auth: unauthenticated | 展示登录表单 |
| A-01 登录 | 登录失败 | 展示错误提示 |
| A-03 赛季管理 | Season: draft / active / closed | 列表中显示状态标签，操作按钮根据状态启用/禁用 |
| A-05 赛事管理 | Tournament: draft / completed / published | 列表中显示状态标签 |
| A-07 Excel 导入 | Upload: pending | 步骤条停在"上传" |
| A-07 Excel 导入 | Upload: parsing | 步骤条停在"解析"，显示加载 |
| A-07 Excel 导入 | Upload: parsed | 步骤条停在"预览"，展示预览表格 |
| A-07 Excel 导入 | Upload: parse_failed | 步骤条"解析"标红，展示错误 |
| A-07 Excel 导入 | Upload: imported | 步骤条全部完成 |
| A-08 排行榜管理 | Ranking: up_to_date | 状态标签显示"最新" |
| A-08 排行榜管理 | Ranking: stale | 状态标签显示"待刷新"，刷新按钮高亮 |
| A-08 排行榜管理 | Ranking: refreshing | 刷新按钮 loading |

## 7. 核心交互说明

### 7.1 排行榜浏览交互

```text
用户进入 → 自动加载排行榜 → 展示列表
                                 │
                    ┌────────────┼────────────┐
                    ↓            ↓            ↓
              输入搜索       选择院系     点击选手姓名
                    │            │            │
                    ↓            ↓            ↓
              过滤列表       过滤列表     弹出积分明细
                                              │
                                              ↓
                                         关闭弹窗
```

### 7.2 Excel 导入交互

```text
选择赛事 → 选择文件 → 点击上传
                          │
                          ↓
                     等待解析完成
                          │
              ┌───────────┼───────────┐
              ↓                       ↓
         解析成功                  解析失败
              │                       │
              ↓                       ↓
         展示预览表格            展示错误信息
              │                       │
     ┌────────┼────────┐              ↓
     ↓        ↓        ↓         重新上传
  修改匹配  忽略行   确认导入
                        │
                        ↓
                   导入成功
                        │
                        ↓
                   生成积分
                        │
                        ↓
                   刷新排行榜
```

### 7.3 赛季激活交互

```text
查看赛季列表 → 点击"激活" → 二次确认弹窗
                                    │
                         ┌──────────┼──────────┐
                         ↓                     ↓
                      确认激活              取消操作
                         │
                         ↓
                  原 active 赛季自动关闭
                  目标赛季变为 active
                         │
                         ↓
                  顶部栏赛季名称更新
```
