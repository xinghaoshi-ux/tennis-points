# 前后端职责边界规范 v1.0

> 本文定义 THA 年度赛事积分系统 MVP 阶段前端与后端的职责边界。
>
> 明确哪些数据由后端提供、哪些动作由前端发起、哪些逻辑归前端、哪些逻辑归后端。

---

## 1. 总体原则

- 后端是数据权威来源和业务规则执行者。
- 前端是交互载体和展示层，不承担业务计算。
- 前端不伪造后端能力，不用本地逻辑替代后端业务结果。
- 后端不关心展示细节，只提供结构化数据。
- 状态迁移由后端控制，前端根据后端返回的状态决定展示。

---

## 2. 前端职责

### 2.1 前端负责的事情

| 类别 | 具体职责 |
|---|---|
| 页面渲染 | 根据后端数据渲染排行榜、表格、表单、弹窗 |
| 交互响应 | 处理用户点击、输入、滚动、筛选等操作 |
| 表单校验 | 前端即时校验（非空、格式、长度），减少无效请求 |
| 文件选择 | 校验文件类型和大小，选择文件后发起上传 |
| 状态展示 | 根据后端返回的 status 字段展示对应 UI 状态 |
| 加载状态 | 管理请求中、成功、失败的 UI 状态 |
| 路由控制 | 管理页面跳转、登录态守卫 |
| 响应式适配 | 桌面端表格布局与移动端卡片布局切换 |
| Token 管理 | 存储、携带、过期检测 Token |
| 错误提示 | 将后端错误码转换为用户可理解的提示文案 |

### 2.2 前端不负责的事情

| 禁止行为 | 说明 |
|---|---|
| 积分计算 | 不在前端计算任何积分值 |
| 排名计算 | 不在前端排序生成排名 |
| 团体积分分摊 | 不在前端做除法分摊 |
| 业务校验 | 不在前端判断赛事等级是否合法、规则是否匹配 |
| 状态迁移 | 不在前端直接修改业务对象状态 |
| 数据持久化 | 不在前端缓存业务数据作为最终结果 |
| 权限判断 | 不在前端做最终权限控制（仅做 UI 隐藏） |

---

## 3. 后端职责

### 3.1 后端负责的事情

| 类别 | 具体职责 |
|---|---|
| 数据存储 | 所有业务数据的持久化和一致性保障 |
| 业务校验 | 赛事等级、组别、成绩、规则匹配等业务规则校验 |
| 积分计算 | 根据规则生成 entries_points 记录 |
| 排行榜计算 | 聚合积分、计算排名、处理并列 |
| Excel 解析 | 读取 Excel 内容、校验数据、生成结构化结果 |
| 状态管理 | 控制赛季、赛事、上传等对象的状态迁移 |
| 认证鉴权 | 验证管理员身份、签发和校验 Token |
| 数据查询 | 提供排行榜、积分明细、赛事列表等查询接口 |
| 错误处理 | 返回结构化错误信息和错误码 |
| 事务保障 | 确保关联操作的原子性 |

### 3.2 后端不负责的事情

| 不负责 | 说明 |
|---|---|
| 页面布局 | 不关心数据如何展示 |
| 交互逻辑 | 不关心用户操作顺序 |
| 前端状态 | 不管理前端加载态、弹窗开关等 UI 状态 |
| 展示排序 | 同积分的展示顺序由后端在查询时处理，但不关心前端如何渲染 |

## 4. 数据提供方（后端 → 前端）

### 4.1 用户端数据

| 数据 | 接口 | 后端提供内容 |
|---|---|---|
| 当前赛季 | GET /api/v1/public/seasons/current | 赛季名称、起止日期 |
| 排行榜 | GET /api/v1/public/rankings | 排名、积分、姓名、年龄、参赛数量、院系（支持分页、搜索、筛选） |
| 选手积分明细 | GET /api/v1/public/players/{id}/points | 基础信息、积分统计、积分明细列表 |
| 院系列表 | GET /api/v1/public/departments | 院系名称列表（用于筛选器） |

### 4.2 管理端数据

| 数据 | 接口 | 后端提供内容 |
|---|---|---|
| 仪表盘统计 | GET /api/v1/admin/dashboard | 选手数、赛事数、积分记录数、最近操作 |
| 赛季列表 | GET /api/v1/admin/seasons | 赛季列表含状态 |
| 选手列表 | GET /api/v1/admin/players | 选手列表（分页、搜索、筛选） |
| 赛事列表 | GET /api/v1/admin/tournaments | 赛事列表含状态 |
| 积分规则 | GET /api/v1/admin/points-rules | 规则列表（按类型分组） |
| 上传记录 | GET /api/v1/admin/uploads/{id} | 上传状态、解析结果、预览数据 |
| 导入预览 | GET /api/v1/admin/uploads/{id}/preview | 逐行预览数据含状态标记 |
| 排行榜 | GET /api/v1/admin/rankings | 排行榜数据 + 刷新状态 |

## 5. 动作发起方（前端 → 后端）

### 5.1 用户端动作

| 动作 | 触发方式 | 请求 |
|---|---|---|
| 加载排行榜 | 页面进入时自动 | GET /api/v1/public/rankings |
| 搜索选手 | 输入姓名后触发 | GET /api/v1/public/rankings?search=xxx |
| 筛选院系 | 选择院系后触发 | GET /api/v1/public/rankings?department=xxx |
| 查看积分明细 | 点击选手姓名 | GET /api/v1/public/players/{id}/points |

### 5.2 管理端动作

| 动作 | 触发方式 | 请求 |
|---|---|---|
| 登录 | 提交登录表单 | POST /api/v1/admin/auth/login |
| 登出 | 点击退出登录 | POST /api/v1/admin/auth/logout |
| 创建赛季 | 提交赛季表单 | POST /api/v1/admin/seasons |
| 激活赛季 | 点击激活按钮 | POST /api/v1/admin/seasons/{id}/activate |
| 关闭赛季 | 点击关闭按钮 | POST /api/v1/admin/seasons/{id}/close |
| 创建选手 | 提交选手表单 | POST /api/v1/admin/players |
| 编辑选手 | 提交编辑表单 | PUT /api/v1/admin/players/{id} |
| 创建赛事 | 提交赛事表单 | POST /api/v1/admin/tournaments |
| 编辑赛事 | 提交编辑表单 | PUT /api/v1/admin/tournaments/{id} |
| 配置规则 | 提交规则表单 | POST /api/v1/admin/points-rules |
| 上传 Excel | 选择文件后上传 | POST /api/v1/admin/uploads |
| 触发解析 | 上传成功后自动 | POST /api/v1/admin/uploads/{id}/parse |
| 轮询解析状态 | 解析中定时查询 | GET /api/v1/admin/uploads/{id} |
| 确认导入 | 点击确认按钮 | POST /api/v1/admin/uploads/{id}/confirm |
| 生成积分 | 导入完成后触发 | POST /api/v1/admin/tournaments/{id}/generate-points |
| 刷新排行榜 | 点击刷新按钮 | POST /api/v1/admin/rankings/refresh |

## 6. 前后端协作边界详解

### 6.1 排行榜展示

| 环节 | 负责方 | 说明 |
|---|---|---|
| 积分聚合 | 后端 | SUM(entries_points) |
| 排名计算 | 后端 | RANK() OVER (ORDER BY total_points DESC) |
| 同积分并列 | 后端 | 并列排名逻辑 |
| 同积分展示排序 | 后端 | 按姓名首字母排序后返回 |
| 分页 | 后端 | 返回指定页数据 |
| 搜索过滤 | 后端 | 按姓名模糊匹配 |
| 院系筛选 | 后端 | 按院系精确匹配 |
| 年龄计算 | 后端 | 根据 birth_date 动态计算 |
| 表格/卡片渲染 | 前端 | 根据屏幕宽度选择布局 |
| 加载态展示 | 前端 | 请求中显示骨架屏或加载动画 |
| 空状态展示 | 前端 | 无数据时显示提示 |

### 6.2 Excel 导入

| 环节 | 负责方 | 说明 |
|---|---|---|
| 文件类型校验 | 前端 | 限制 .xlsx / .xls |
| 文件大小校验 | 前端 | 限制 10MB |
| 文件上传 | 前端发起 | multipart/form-data |
| 文件存储 | 后端 | 保存到文件系统 |
| Excel 读取 | 后端 | openpyxl / pandas |
| 数据校验 | 后端 | 字段完整性、格式、业务规则 |
| 选手匹配 | 后端 | 姓名匹配已有选手 |
| 预览数据生成 | 后端 | 结构化预览 + 行级状态 |
| 预览展示 | 前端 | 渲染预览表格 |
| 异常行标记 | 后端提供标记，前端展示 | 后端标记行状态，前端标红 |
| 手动匹配选手 | 前端发起 | 管理员选择正确选手 |
| 确认导入 | 前端发起 | 后端执行写入 |
| 写入 event_results | 后端 | 事务写入 |

### 6.3 积分生成

| 环节 | 负责方 | 说明 |
|---|---|---|
| 触发生成 | 前端发起 | 点击按钮 |
| 规则匹配 | 后端 | 根据赛事级别、组别、成绩查找规则 |
| 单项积分计算 | 后端 | 直接取规则积分值 |
| 团体积分分摊 | 后端 | round(团队总积分 / 有效队员人数) |
| 远程奖补计算 | 后端 | 根据跨省/跨境标记加积分 |
| 写入 entries_points | 后端 | 事务写入 |
| 生成结果展示 | 前端 | 展示成功/失败提示 |

### 6.4 认证与权限

| 环节 | 负责方 | 说明 |
|---|---|---|
| 凭证验证 | 后端 | 校验用户名密码 |
| Token 签发 | 后端 | 生成 JWT |
| Token 存储 | 前端 | localStorage 或内存 |
| Token 携带 | 前端 | 请求头 Authorization |
| Token 校验 | 后端 | 每次请求验证 |
| 过期检测 | 前端 + 后端 | 前端检测 401 响应，跳转登录 |
| 路由守卫 | 前端 | 无 Token 时拦截跳转登录页 |
| 接口权限 | 后端 | 管理端接口要求有效 Token |

## 7. 前端表单校验 vs 后端业务校验

前端校验是用户体验优化，后端校验是安全保障。两者不可互相替代。

| 校验类型 | 前端 | 后端 |
|---|---|---|
| 非空校验 | 做 | 做 |
| 格式校验（日期、数字） | 做 | 做 |
| 长度限制 | 做 | 做 |
| 文件类型 | 做 | 做 |
| 文件大小 | 做 | 做 |
| 赛事等级合法性 | 不做 | 做 |
| 积分规则匹配 | 不做 | 做 |
| 选手是否存在 | 不做 | 做 |
| 赛季状态是否允许操作 | 不做 | 做 |
| 重复数据检测 | 不做 | 做 |
| 团体队员人数校验 | 不做 | 做 |

## 8. 接口错误处理分工

| 错误类型 | HTTP 状态码 | 后端职责 | 前端职责 |
|---|---|---|---|
| 参数校验失败 | 422 | 返回字段级错误信息 | 展示在对应表单字段下方 |
| 业务规则冲突 | 409 | 返回冲突原因 | 展示全局提示 |
| 未认证 | 401 | 返回 401 | 清除 Token，跳转登录 |
| 权限不足 | 403 | 返回 403 | 展示"无权限"提示 |
| 资源不存在 | 404 | 返回 404 | 展示"数据不存在"提示 |
| 服务器错误 | 500 | 记录日志，返回通用错误 | 展示"系统异常，请稍后重试" |
| 请求超时 | 前端判断 | 无 | 展示"请求超时"+ 重试按钮 |

## 9. 数据刷新策略

| 场景 | 策略 | 说明 |
|---|---|---|
| 用户端排行榜 | 进入页面时请求 | 不做定时轮询，用户手动刷新页面获取最新 |
| 管理端列表 | 操作后刷新 | 新增/编辑/删除后重新请求列表 |
| Excel 解析状态 | 轮询 | 解析中每 3 秒查询一次状态，直到终态 |
| 排行榜刷新状态 | 轮询 | 刷新中每 3 秒查询一次，直到完成 |
| 积分生成状态 | 轮询 | 生成中每 3 秒查询一次，直到完成 |
